# Sprint 2 — DWH basics: staging/core/mart

## Цель
Спроектировать учебный DWH на основе raw-данных.

1. Описать модель: raw → dwh → mart.

Слой RAW:
regions- таблица фиксирует базовый перечень регионов с их уникальными кодами и официальными наименованиями
departments- справочник который содержит реестр районных и городских отделов образования с обязательном привязкой к конкретному региону
academic_periods- таблица делит учебные года на конкретные периоды обучения с фиксацией их дат и начала окончания
bi_targets- специфический справочник хранит плановые целевые показатели успеваемости и посещаемости для натсройки план факт релиза
schools- паспорт содержит данные по школам с указанием бин, языка обучения, типа местности, статус активности и даты открытия 
school_name_aliases- таблица собирает альтернативные и варианты названия школ 
teachers- реестр фиксирует персональные карточки учителей, дата приема на работу, статус активности, 
subjects-  справочник хранит каталог учебных предметов и их предметных групп
groups- список школьных классов с указанем буквы класса, грейда, года обучения, смены и статус активности
students- персональные данные учеников, дата принятия, статус активности 
student_marks- журнал регистрирует текущие оценки учеников, указывая тип работы и даты выставления
exam_results- оцненки по экзаменам, номера попыток сдачи, максимальные лимиты баллов
student_attendance- ежедневная посещаемость, статус присутствия, точное время опоздания
expected_ dq_issues- перечень и описание ожидаемых ошибок качества данных в текущем датасете

2. Слой DWH
Таблицы измерений (Справочники контекста)
dim_region- очищенная таблица регионов гарантирует отсутствие текстовых дублей и хранит проверенные географические коды областей.
dim_department- изолированный справочник районов распределяет отделы образования по их территориальной принадлежности.
dim_school- консолидированная таблица объединяет профили школ с данными их департаментов и регионов через левые соединения.
dim_subject- нормализованный каталог предметов исключает дублирование названий дисциплин при связывании с фактами.
dim_group- измерение классов структурирует учебные группы для фильтрации показателей по параллелям и сменам.
dim_student- эталонный справочник учеников хранит верифицированные персональные данные без системного мусора и повторов.

Таблицы фактов 
fact_marks- таблица собирает все события выставления текущих оценок и результатов экзаменов в единую атомарную структуру.
fact_attendance- лог аккумулирует факты посещаемости и очищенные метрики времени опозданий для расчета дисциплины.

3.Mart- витрина агрегированных показателей успеваемости в разрезе: Регион -> Департамент -> Школа -> Класс -> Предмет -> Четверть.
    Витрина рассчитывает два ключевых KPI для Министерства образования:
    1.Качество знаний (`quality_of_knowledge_pct`)— процент учеников, завершивших период на оценки `4` и `5`
    2.Успеваемость (`passing_rate_pct`)— процент учеников, успешно освоивших программу без двоек (включая оценки `3, 4, 5`).

## Ключевые бизнес формулы витрины

* **Процент качества знаний:**
    $$Quality\ of\ Knowledge\ \% = \frac{Quality\ Students\ Count\ (оценки\ 4\ and\ 5)}{Total\ Graded\ Students} \times 100$$

* **Процент успеваемости (доля успевающих учеников):**
    $$Passing\ Rate\ \% = \frac{Passing\ Students\ Count\ (оценки\ 3,\ 4\ and\ 5)}{Total\ Graded\ Students} \times 100$$

## Выводы витрины

Витрина данных `mart.mart_quality_of_knowledge` спроектирована как инструмент для принятия управленческих решений. На основе собираемых метрик руководство школ и управлений образования может формировать следующие ключевые выводы:

1. **Выявление проблемных предметов и классов**
   Сравнение показателей качества знаний (`quality_of_knowledge_pct`) по разным предметам внутри одного класса позволяет точечно находить аномалии. Например, если в 5 А классе качество знаний по русской литературе составляет 85%, а по математике падает до 20%, это сигнализирует о системной проблеме в усвоении точных наук или о необходимости методической поддержки учителя математики.

2. **Оценка скрытой неуспеваемости (Разрыв между качеством и успеваемостью)**
   Метрика успеваемости `passing_rate_pct` показывает долю учеников без двоек. Если в школе показатель успеваемости близок к 100%, но при этом качество знаний (`quality_of_knowledge_pct`) находится на уровне 15-20%, это означает, что подавляющее большинство детей учится строго на тройки. Школа не имеет провалов, но находится в зоне образовательной стагнации. Витрина позволяет оперативно подсветить этот разрыв.

3. **Мониторинг динамики по четвертям**
   Использование разреза по учебным периодам (`quarter_number`) помогает отслеживать сезонность обучения. Традиционно третья четверть является самой длинной и сложной. Витрина помогает зафиксировать, насколько сильно падает качество знаний к весне, чтобы администрация могла скорректировать учебную нагрузку или расписание.

4. **Анализ эффективности работы департаментов**
   Агрегация данных до уровня `department_name` позволяет сравнивать успешность образовательных программ между целыми районами или округами, выявляя те управления, которые нуждаются в дополнительном финансировании или обновлении кадрового состава.
